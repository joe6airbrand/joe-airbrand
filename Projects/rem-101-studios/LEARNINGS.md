# LEARNINGS.md — rem-101-studios

Running log of generalizable learnings about how the user makes
architectural, design, and process decisions on this project.

**Read this file at the start of every new chat.** Emphasize the most
recent entries (top of the list). When a newer learning supersedes an
older one, edit or remove the older entry so the log stays internally
consistent — do not let stale guidance accumulate.

Each entry should be:

* **Generalizable** — applies beyond the single moment it was learned.
* **About *how* the user thinks** — preferences, taste, working style —
  not implementation trivia (those go in code or `DESIGN.md`).
* **Concise** — one or two sentences plus a short rationale if useful.

Format:

```
### YYYY-MM-DD — short title
One-line statement of the learning.
Optional brief rationale or example.
```
